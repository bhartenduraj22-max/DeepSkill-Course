# SOLID Principles
SOLID is Collections of 5 rules used to write better code. In simple guidline to write java Code.
### Helps in:
Easy to Understand
Ease to maintain
Easy to modify
Easy to test
### Benifits :
Cleaner code
fewer Bug
Easier testing
## SOLID
### S : Single Responsibility Principle
One class should have only one job.
### O : Open Closed Principle
Add new features without changing existing code.
### L : Liskov Substitution Principle
Child classes should replace parent classes correctly.
### I :  Interface Segregation Principle
Create small interfaces instead of large interfaces.
### D :  Dependency Inversion Principle
Depend on abstractions, not concrete classes.
